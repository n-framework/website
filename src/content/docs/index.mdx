## What is NFramework?

NFramework is a compile-time-first application framework and workspace standard for building clean architecture services across .NET, Go, and Rust. It removes the runtime bloat, reflection-heavy startup cost, and inconsistent service structure found in traditional enterprise frameworks.

The `nfw` CLI gives teams a fast and opinionated way to scaffold services, enforce architecture boundaries, and generate boilerplate safely at compile time.

## Key Features

- **CLI-driven workflow** — Create workspaces, scaffold services, and validate architecture with a single tool.
- **Clean Architecture enforced** — Layer boundaries checked automatically, not just documented.
- **Compile-time generation** — DI registration and route discovery at build time, not runtime.
- **Zero-dependency core** — Framework packages have no external library dependencies.
- **Replaceable adapters** — Swap persistence, logging, and validation libraries without touching domain code.
- **Polyglot ready** — One workspace standard across .NET, Go, and Rust.

## Quick Start

```bash
# Install
git clone https://github.com/n-framework/n-framework.git
cd n-framework && make build

# Create a workspace
nfw new my-workspace

# Add a service
cd my-workspace && nfw add service my-api
```

## Product Plan

- **Beta scope**: .NET-first, execution-focused with CLI, generators, and architecture validation.
- **Product vision**: Polyglot ecosystem across .NET, Go, and Rust with contract-driven development.
- **MCP support**: Tool and agent integrations for the workspace and CLI ecosystem.
- **Non-negotiables**: Clean Architecture, Native AOT, and CLI-driven automation.
